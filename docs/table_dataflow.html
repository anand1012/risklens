Pipeline execution order (daily): bronze_rates.py → bronze_prices.py → bronze_trades.py → bronze_synthetic.py → silver_transform.py (clean all 4 bronze tables → silver.rates/prices/trades/risk_outputs + updates catalog.quality_scores) → silver_enrich.py (join silver tables → silver.positions + silver.risk_enriched) → gold_aggregate.py (compute 8 gold tables + updates catalog.sla_status)

Quality gate: Records that fail null checks or bounds checks in silver_transform.py are written to bronze.quarantine_r (never deleted). DTCC SDR: Returns HTTP 404 in current environment — bronze.trades_r and silver.trades have 0 rows; gold.trade_positions has 0 rows. Synthetic risk model fills in VaR/ES/P&L data for all 6 desks.
